## 1. Что понадобится

### Сервер (VPS/VDS)
- **ОС**: Ubuntu 22.04 LTS (рекомендуется)
- **RAM**: минимум 1 ГБ (лучше 2 ГБ)
- **CPU**: 1 ядро минимум
- **Диск**: 10 ГБ минимум
- **Открытые порты**: 22 (SSH), 80 (HTTP), 443 (HTTPS)

Подойдёт любой хостинг: Timeweb, Reg.ru, Selectel, Hetzner и т.д.

### На вашем компьютере
- **SSH-клиент**: на Windows — [PuTTY](https://putty.org) или Windows Terminal
- **SFTP-клиент** для загрузки файлов: [FileZilla](https://filezilla-project.org) (бесплатно)
- Архив проекта (папка `html` из ZIP-архива БРО)

### Домен
- Любой домен, указывающий A-записью на IP вашего сервера
- Если домена нет — пока можно использовать IP-адрес сервера

## 2. Подключение к серверу

Откройте терминал (Windows: нажмите Win+R, введите `cmd`) и выполните:

```bash
ssh root@ВАШ_IP_АДРЕС
```

Например: `ssh root@185.100.200.50`

Введите пароль (при вводе пароля символы не отображаются — это нормально, просто нажмите Enter).

**Первое, что нужно сделать** — обновить систему:

```bash
apt update && apt upgrade -y
```

Это может занять 2–5 минут.

## 3. Установка Node.js

Проект требует **Node.js версии 20** (LTS).

```bash
# Добавляем официальный репозиторий Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -

# Устанавливаем Node.js
apt install -y nodejs

# Проверяем версии (должно показать v20.x.x и 10.x.x)
node --version
npm --version
```

Устанавливаем PM2 — менеджер процессов, который будет запускать сайт и перезапускать его при падении:

```bash
npm install -g pm2
```

## 4. Установка PostgreSQL

PostgreSQL — база данных, в которой хранятся новости, аналитика, заявки.

```bash
# Устанавливаем PostgreSQL
apt install -y postgresql postgresql-contrib

# Запускаем и добавляем в автозагрузку
systemctl start postgresql
systemctl enable postgresql

# Проверяем статус (должно быть "active (running)")
systemctl status postgresql
```

Нажмите **Q** чтобы выйти из просмотра статуса.

## 5. Создание базы данных

Переходим в режим пользователя postgres:

```bash
sudo -u postgres psql
```

Вы увидите строку `postgres=#`. Вводим команды по очереди:

```sql
-- Создаём пользователя для сайта (замените ПРИДУМАЙТЕ_ПАРОЛЬ на свой пароль)
CREATE USER bro_user WITH PASSWORD 'ПРИДУМАЙТЕ_ПАРОЛЬ';

-- Создаём базу данных
CREATE DATABASE bro_db OWNER bro_user;

-- Даём права
GRANT ALL PRIVILEGES ON DATABASE bro_db TO bro_user;

-- Выходим из psql
\q
```

**Запишите** пользователя, пароль и имя базы — они понадобятся в следующем шаге.

Пример строки подключения (DATABASE_URL):
```
postgresql://bro_user:ПРИДУМАЙТЕ_ПАРОЛЬ@localhost:5432/bro_db
```

## 6. Загрузка проекта на сервер

### Вариант А: через FileZilla (рекомендуется для начинающих)

1. Откройте FileZilla
2. В поле **Хост**: `sftp://ВАШ_IP`
3. **Имя пользователя**: `root`
4. **Пароль**: пароль от сервера
5. **Порт**: `22`
6. Нажмите **Быстрое соединение**
7. В правой панели (сервер) перейдите в `/var/www/`
8. Перетащите папку `html` из архива в `/var/www/`
9. Переименуйте папку: правой кнопкой → Переименовать → `bro`

Итог: файлы проекта должны быть в `/var/www/bro/`.

### Вариант Б: через git (если проект в репозитории)

```bash
cd /var/www
git clone https://github.com/ВАШ_АККАУНТ/bro.git bro
```

## 7. Настройка переменных окружения

Переходим в папку проекта:

```bash
cd /var/www/bro
```

**Шаг 7.1 — Генерируем SESSION_SECRET** (нужен только встроенный Node.js, устанавливать ничего не нужно):

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

Скопируйте вывод — он понадобится через секунду.

**Шаг 7.2 — Создаём файл настроек:**

```bash
nano .env
```

Откроется текстовый редактор. Вставьте следующее (замените значения на свои):

```env
# База данных (замените bro_user и ПРИДУМАЙТЕ_ПАРОЛЬ на свои из шага 5)
DATABASE_URL="postgresql://bro_user:ПРИДУМАЙТЕ_ПАРОЛЬ@localhost:5432/bro_db"
DIRECT_URL="postgresql://bro_user:ПРИДУМАЙТЕ_ПАРОЛЬ@localhost:5432/bro_db"

# Секрет для сессий — вставьте результат команды из шага 7.1
SESSION_SECRET="сгенерированная_строка_из_шага_7_1"

# Хеш пароля администратора — заполним в шаге 8 после npm install
ADMIN_PASSWORD_HASH="placeholder"

# Адрес вашего сайта (без слеша в конце)
NEXT_PUBLIC_BASE_URL="https://ваш-домен.ru"
```

Сохраните: **Ctrl+O**, Enter, **Ctrl+X**.

## 8. Установка зависимостей, настройка пароля и сборка

**Шаг 8.1 — Устанавливаем зависимости:**

```bash
cd /var/www/bro
npm install
```

**Шаг 8.2 — Генерируем хеш пароля администратора:**

Придумайте пароль для входа в `/admin` и выполните (замените `ВАШ_ПАРОЛЬ`):

```bash
node -e "const b=require('bcryptjs'); b.hash('ВАШ_ПАРОЛЬ', 10).then(console.log)"
```

Скопируйте вывод (строка начинается с `$2a$10$...`).

**Шаг 8.3 — Вставляем хеш в .env:**

```bash
nano .env
```

Найдите строку `ADMIN_PASSWORD_HASH="placeholder"` и замените `placeholder` на скопированный хеш. Сохраните: **Ctrl+O**, Enter, **Ctrl+X**.

**Запомните пароль** — он нужен для входа в `/admin`.

**Шаг 8.4 — Применяем схему и заполняем базу:**

```bash
# Создаёт все таблицы в базе данных
npx prisma db push

# Заполняем начальными данными (первая новость, баннер, релиз v1.0.0)
npx prisma db seed
```

**Шаг 8.5 — Собираем проект:**

```bash
npm run build
```

Сборка занимает 1–3 минуты. Если завершилась без ошибок — всё готово к запуску.

## 9. Запуск через PM2

PM2 — менеджер процессов. Он:
- Запускает сайт в фоне
- Автоматически перезапускает при падении
- Сохраняет логи

```bash
# Создаём папку для логов
mkdir -p /var/log/bro

cd /var/www/bro

# Запускаем приложение через готовый конфиг (рекомендуется)
pm2 start pm2.config.js

# Или альтернативный способ без конфига:
# pm2 start npm --name "bro" -- start

# Сохраняем список процессов (чтобы PM2 восстановил их после перезагрузки сервера)
pm2 save

# Добавляем PM2 в автозагрузку
pm2 startup
```

Последняя команда выведет строку вида `sudo env PATH=... pm2 startup ...`. **Скопируйте её и выполните**.

### Полезные команды PM2

```bash
pm2 status          # статус всех процессов
pm2 logs bro        # просмотр логов в реальном времени (Ctrl+C для выхода)
pm2 restart bro     # перезапустить сайт
pm2 stop bro        # остановить сайт
```

После запуска сайт доступен на порту **3000**: `http://ВАШ_IP:3000`

## 10. Настройка Nginx

Nginx — веб-сервер, который принимает запросы на 80/443 порт и перенаправляет их к нашему приложению на порту 3000.

```bash
# Устанавливаем Nginx
apt install -y nginx

# Запускаем и добавляем в автозагрузку
systemctl start nginx
systemctl enable nginx
```

Создаём конфигурацию для нашего сайта:

```bash
nano /etc/nginx/sites-available/bro
```

Вставьте следующее (замените `ваш-домен.ru` на реальный домен или IP):

```nginx
server {
    listen 80;
    server_name ваш-домен.ru www.ваш-домен.ru;

    # Максимальный размер загружаемых файлов (для изображений)
    client_max_body_size 10M;

    # Отдаём статические файлы Next.js напрямую через Nginx (быстрее)
    location /_next/static/ {
        alias /var/www/bro/.next/static/;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Загруженные изображения (если используется локальное хранилище)
    location /uploads/ {
        alias /var/www/bro/public/uploads/;
        expires 30d;
        add_header Cache-Control "public";
    }

    # Всё остальное — проксируем к Next.js
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Сохраните: **Ctrl+O**, Enter, **Ctrl+X**.

Активируем конфигурацию:

```bash
# Создаём символическую ссылку
ln -s /etc/nginx/sites-available/bro /etc/nginx/sites-enabled/

# Проверяем синтаксис конфигурации (должно быть "syntax is ok")
nginx -t

# Перезапускаем Nginx
systemctl reload nginx
```

Теперь сайт доступен по адресу `http://ваш-домен.ru`.

## 11. SSL-сертификат (HTTPS)

Бесплатный SSL от Let's Encrypt. Нужен реальный домен, указывающий на IP сервера.

```bash
# Устанавливаем Certbot
apt install -y certbot python3-certbot-nginx

# Получаем сертификат и автоматически настраиваем Nginx
certbot --nginx -d ваш-домен.ru -d www.ваш-домен.ru
```

Certbot спросит email (для уведомлений об истечении) и попросит принять условия. После этого он автоматически:
- Получит сертификат
- Обновит конфигурацию Nginx для работы через HTTPS
- Настроит автоматическое обновление сертификата (каждые 90 дней)

Проверяем автообновление:
```bash
certbot renew --dry-run
```

Сайт теперь доступен по `https://ваш-домен.ru`.

## 13. Хранилище изображений

При загрузке изображений в новостях проект автоматически выбирает бэкенд:

### По умолчанию: локальные файлы
Никакой настройки не нужно. Изображения сохраняются в `/var/www/bro/public/uploads/`.

**Важно**: эти файлы не попадают в git. При обновлении проекта они сохраняются, но если вы переустанавливаете сервер с нуля — нужно их перенести вручную.

### S3-совместимое хранилище (рекомендуется для продакшна)

Yandex Cloud Object Storage — популярный выбор для российских проектов.

1. Зайдите в консоль Yandex Cloud → Object Storage
2. Создайте бакет (например `bro-uploads`), выберите **Публичный доступ: Включён**
3. В разделе **Сервисные аккаунты** создайте аккаунт с ролью `storage.uploader`
4. Создайте для него статический ключ доступа
5. Добавьте в `/var/www/bro/.env`:

```env
S3_ENDPOINT="https://storage.yandexcloud.net"
S3_REGION="ru-central1"
S3_ACCESS_KEY_ID="идентификатор_ключа"
S3_SECRET_ACCESS_KEY="секретный_ключ"
S3_BUCKET_NAME="bro-uploads"
```

Перезапустите приложение: `pm2 restart bro`

## 15. Частые ошибки

### Сайт не открывается по домену
1. Проверьте, что DNS A-запись домена ведёт на правильный IP: `nslookup ваш-домен.ru`
2. Проверьте Nginx: `nginx -t && systemctl status nginx`
3. Проверьте что PM2 запущен: `pm2 status`
4. Проверьте логи: `pm2 logs bro --lines 50`

### Ошибка "Cannot connect to database"
1. Проверьте, что PostgreSQL запущен: `systemctl status postgresql`
2. Проверьте строку подключения в `.env` — пользователь, пароль, имя БД
3. Проверьте, что база создана: `sudo -u postgres psql -c "\l"`

### Ошибка при `npm run build`
- Убедитесь, что файл `.env` существует и заполнен
- Проверьте версию Node.js: `node --version` (нужна v20)
- Посмотрите точный текст ошибки — обычно указывает на проблему

### Страница `/admin` показывает 404 или ошибку
1. Убедитесь, что сборка прошла успешно
2. Проверьте переменные окружения: `SESSION_SECRET` и `ADMIN_PASSWORD_HASH` не должны быть пустыми

### Изображения не загружаются в новостях
1. Проверьте права на папку uploads:
   ```bash
   mkdir -p /var/www/bro/public/uploads/news
   chown -R www-data:www-data /var/www/bro/public/uploads
   chmod -R 755 /var/www/bro/public/uploads
   ```
2. Или подключите S3-хранилище (см. раздел 13)

### PM2 не восстанавливается после перезагрузки сервера
```bash
pm2 save
pm2 startup
# выполните выведенную команду с sudo
```

## Быстрая шпаргалка команд

```bash
# Статус приложения
pm2 status

pm2 logs bro

sudo -u postgres psql -c "SELECT count(*) FROM pg_stat_activity WHERE datname='bro_db';"

```

```
├── public/

│   └── schema.prisma  ← схема базы данных
└── package.json
```
